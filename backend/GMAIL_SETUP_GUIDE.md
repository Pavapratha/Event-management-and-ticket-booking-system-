# Gmail SMTP Configuration Guide for Nodemailer

## Problem: Error 535 5.7.8 BadCredentials

If you're seeing this error:
```
Invalid login: 535-5.7.8 Username and Password not accepted
Code: EAUTH
SMTP Response: 535 5.7.8 BadCredentials
```

**The issue**: You're using your actual Gmail password instead of a **Gmail App Password**.

---

## Solution: Generate and Use a Gmail App Password

### What is an App Password?

An **App Password** is a 16-character password generated by Google specifically for third-party apps like your Node.js application. It's more secure than using your actual Gmail password because:
- It's unique to the app
- It can be revoked anytime
- It only works for mail access, not other account functions
- It requires 2-Step Verification to be enabled

### Step 1: Enable 2-Step Verification

1. Go to **[https://myaccount.google.com/](https://myaccount.google.com/)**
2. Click **"Security"** in the left sidebar
3. Under "How you sign in to Google", find **"2-Step Verification"**
4. Click on it and follow the prompts to enable it
5. Google will ask you to verify your identity using a recovery method

**Why?** Gmail requires 2-Step Verification before you can generate App Passwords.

### Step 2: Generate an App Password

1. Go to **[https://myaccount.google.com/apppasswords](https://myaccount.google.com/apppasswords)**
   - If you don't see "App passwords", make sure:
     - 2-Step Verification is enabled (Step 1)
     - You're signed in with the correct Gmail account
2. Select **"Mail"** from the first dropdown
3. Select **"Windows Computer"** (or your device type) from the second dropdown
4. Click **"Generate"**
5. Google will display a 16-character password (example: `abcd efgh ijkl mnop`)
6. **Copy this password** (including spaces, but we'll remove them in the .env file)

### Step 3: Update Your .env File

Edit `backend/.env` and update the `SMTP_PASSWORD` field:

**Before (❌ Wrong - using actual Gmail password):**
```env
SMTP_PASSWORD=your_actual_gmail_password
```

**After (✅ Correct - using 16-character App Password):**
```env
SMTP_PASSWORD=abcdefghijklmnop
```

**Important**: Remove spaces from the App Password. If Google gives you `abcd efgh ijkl mnop`, use `abcdefghijklmnop`.

### Complete .env Email Configuration

```env
# Gmail SMTP Configuration
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_SECURE=false
SMTP_USER=your-email@gmail.com
SMTP_PASSWORD=yourcodefrommyaccountgooglecomapppasswords
SMTP_FROM_EMAIL=noreply@eventmanagement.com
```

### Step 4: Restart Your Server

After updating the `.env` file:

```bash
# Stop the server (Ctrl+C)
# Restart it
npm start
```

You should see this message:
```
=======================================================================
✅ EMAIL TRANSPORTER VERIFICATION SUCCESSFUL
=======================================================================
```

## Nodemailer Configuration Details

### Why We Use These Settings

| Setting | Value | Why? |
|---------|-------|------|
| **SMTP_HOST** | `smtp.gmail.com` | Gmail's SMTP server |
| **SMTP_PORT** | `587` | TLS port (encrypted connection) |
| **SMTP_SECURE** | `false` | false for port 587, true for port 465 |
| **SMTP_USER** | Your full Gmail address | Your Gmail account |
| **SMTP_PASSWORD** | 16-char App Password | Security-verified by Google |

### Port Options

- **Port 587** (Recommended): Uses StartTLS encryption
  - Set `SMTP_SECURE=false`
  - Connection starts unencrypted, then upgrades to TLS
  
- **Port 465** (Alternative): Uses SMTPS (implicit SSL)
  - Set `SMTP_SECURE=true`
  - Connection is encrypted from the start

## Error Troubleshooting

### Error: 535 5.7.8 BadCredentials
**Cause**: Wrong password (using actual Gmail password)
**Fix**: 
1. Generate Gmail App Password from `https://myaccount.google.com/apppasswords`
2. Update `SMTP_PASSWORD` in `.env`
3. Restart server

### Error: ECONNREFUSED (Connection Refused)
**Cause**: Can't connect to Gmail SMTP server
**Fix**:
- Verify `SMTP_HOST=smtp.gmail.com`
- Verify `SMTP_PORT=587`
- Check firewall/network settings
- Try a different network

### Error: ENOTFOUND (Host Not Found)
**Cause**: DNS resolution failed
**Fix**:
- Check internet connection
- Try `ping smtp.gmail.com` in terminal
- May be DNS/network issue

### Error: ETIMEDOUT (Connection Timeout)
**Cause**: Server not responding in time
**Fix**:
- Network might be slow
- Firewall might be blocking
- Try again or contact network admin

## Security Notes

✅ **App Passwords are secure because:**
- They're unique to your app
- They only work for email/calendar access
- They don't give access to your full account
- You can revoke them anytime
- 2-Step Verification requirement prevents misuse

❌ **Never do these:**
- Don't use your actual Gmail password in code
- Don't enable "Less secure app access" (deprecated)
- Don't commit `.env` with real credentials to Git
- Don't share your App Password

## Production Recommendations

For production use:

1. **Use Environment Variables** (already doing this)
   ```bash
   SMTP_USER=your-email@gmail.com
   SMTP_PASSWORD=your-16-char-app-password
   ```

2. **Consider sending limits**
   - Gmail has sending limits (~100 emails/hour initially)
   - For more, use SendGrid, Mailgun, or AWS SES

3. **Add Email Retry Logic**
   - Implement exponential backoff
   - Queue failed emails for retry

4. **Monitor Email Delivery**
   - Log all successful/failed sends
   - Use Gmail SMTP logs for debugging

## Still Having Issues?

1. **Check all 3 things:**
   - 2-Step Verification is ENABLED
   - You're using an App Password (not Gmail password)
   - App Password is exactly 16 characters

2. **Try regenerating:**
   - Go to `https://myaccount.google.com/apppasswords`
   - Generate a new App Password
   - Update `.env` with the new one
   - Restart server

3. **Check logs:**
   - Backend logs show detailed error info
   - Look for "EAUTH", "BadCredentials", or specific error codes

4. **Test connectivity:**
   ```bash
   # From terminal, test connection to Gmail SMTP
   telnet smtp.gmail.com 587
   ```
